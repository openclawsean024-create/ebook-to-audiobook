# 電子書轉有聲書 - 專案規格 / 目前進度

## 專案目標
把電子書內容轉成可朗讀、可分段控制、最終可匯出音檔的有聲書流程。

---

## 目前版本：v4.0 Demo
目前已完成一個 **純前端、可直接部署的靜態版 Demo**：

### 已完成功能
1. **電子書上傳與解析**
   - 支援格式：PDF / EPUB / TXT
   - PDF：使用 pdf.js 進行瀏覽器端文字抽取
   - EPUB：使用 JSZip 解壓 EPUB，讀取 OPF / spine / 章節 HTML 內容
   - TXT：直接讀取文字

2. **書籍內容整理**
   - 自動清理空白與換行
   - 顯示檔名、格式、標題、作者（EPUB 有 metadata 時會顯示）
   - 提供前 1800 字預覽

3. **有聲書播放流程（Demo 版）**
   - 使用瀏覽器 Web Speech API 進行朗讀
   - 將全文自動切成多個段落 segment
   - 顯示段落數、目前段落、段落字數、預估閱讀時間
   - 支援：播放 / 暫停 / 繼續 / 停止 / 上一段 / 下一段
   - 可做語音試聽

4. **下載能力（現階段）**
   - 可下載整理後的純文字內容
   - 音檔下載目前仍是說明模式，尚未輸出真正 MP3

---

## 目前限制
因為現在是 **純前端靜態網站**，所以有幾個硬限制：

1. **不能真正匯出 MP3**
   - Web Speech API 只能播放，大多數瀏覽器不能直接把語音輸出成檔案

2. **語音品質依使用者裝置而異**
   - 不同作業系統 / 瀏覽器可用的 voice 不同
   - 中文聲音名稱也不一定完全一致

3. **大檔案穩定度有限**
   - 超長 PDF / EPUB 在瀏覽器端處理時，速度與記憶體會受限制

4. **沒有雲端任務與背景轉檔**
   - 關閉頁面後，流程就中斷

---

## 下一階段：真正可商用的後端版
若要從 Demo 升級成真正的「轉檔器」，下一階段應加入後端：

### 後端建議
- **FastAPI** 或 **Flask**
- 上傳檔案到後端
- 伺服器端抽取電子書文字
- 將文字依章節切段
- 呼叫 TTS 引擎批次產生音訊檔
- 合併為 MP3 / M4B
- 回傳下載連結

### TTS 建議
- **Edge TTS**：免費、自然、適合 MVP
- **Azure TTS**：品質高、語音選擇多
- **ElevenLabs**：最自然，但成本較高

### 下一階段功能清單
1. 真正輸出 MP3 / M4B
2. 章節列表與章節跳轉
3. 背景排程轉檔
4. 雲端儲存與分享連結
5. 多語音批次生成
6. 封面、書名、作者 metadata 寫入音檔
7. 斷點續轉

---

## 現況總結
- **現在可用來 demo 使用者流程與前端體驗**
- **下一步最值得投資的是：接後端 TTS，讓它真的產出音檔**

如果要繼續開發，建議優先順序：
1. FastAPI 後端
2. Edge TTS 產檔
3. MP3 合併下載
4. Vercel + API 部署
